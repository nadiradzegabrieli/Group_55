1)რა არის HTTP?
HTTP (HyperText Transfer Protocol) — არის ქსელური პროტოკოლი, რომელიც გამოიყენება ვებ-ბრაუზერსა და სერვერს შორის ინფორმაციის გადაცემისთვის. სწორედ HTTP-ის მეშვეობით ხდება HTML გვერდების, ფაილების, სურათების და API მონაცემების გამოთხოვა.
HTTP მუშაობს request-response პრინციპით: მომხმარებელი აგზავნის request-ს (მოთხოვნას), ხოლო სერვერი პასუხობს response-ით (პასუხით).


2)რა არის HTTP Request და Request Methods?
HTTP Request — არის მოთხოვნა, რომელსაც აგზავნის კლიენტი (მაგ. ბრაუზერი ან JavaScript კოდი) სერვერისკენ, გარკვეული ინფორმაციის მოთხოვნის ან მონაცემის გაგზავნის მიზნით.
Request-ის სტრუქტურა მოიცავს:
- მეთოდს (GET, POST და სხვა)
- მისამართს (URL)
- სათაურებს (Headers)
- საჭიროების შემთხვევაში — სხეულს (Body)
Request Methods — ეს არის მოქმედებები, რომლითაც კლიენტი განსაზღვრავს რას აკეთებს:
- GET — გამოთხოვა მონაცემების; გამოიყენება ინფორმაციის მისაღებად, მონაცემების შეცვლის გარეშე.
- POST — მონაცემების გაგზავნა სერვერზე (მაგ. ფორმის გაგზავნა); გამოიყენება ახალი ობიექტების შესაქმნელად.
- PUT — სრული განახლება; გამოიყენება მონაცემის მთლიანი შეცვლისთვის.
- DELETE — მონაცემის წაშლა.
- PATCH — ნაწილობრივი განახლება.


3)რა არის HTTP Response და Response Codes
HTTP Response — ეს არის სერვერის მიერ გამოგზავნილი პასუხი მომხმარებლის request-ზე. Response შეიცავს:
- სტატუს კოდს (Status Code)
- სათაურებს (Headers)
- მონაცემს (Body)

4)Codes

200 - OK - მოთხოვნა წარმატებით შესრულდა.
201 - Created - ახალი ობიექტი წარმატებით შეიქმნა
400 - Bad Request - მოთხოვნა არასწორია.
401 - Unauthorized - საჭიროა ავტორიზაცია.
403 - Forbidden - დაშვება აკრძალულია.
404 - Not Found - რესურსი ვერ მოიძებნა.
500 - Internal Server Error - სერვერზე მოხდა შეცდომა.
